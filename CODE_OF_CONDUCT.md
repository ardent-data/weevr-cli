# Code of Conduct

## Principle

This project operates on a single expectation: treat others with basic professional respect.

## Standards

**Expected:**

- Focus discussion on the work, the design, and the code
- Give constructive, specific feedback — critique the approach, not the person
- Accept that reasonable people can disagree on technical tradeoffs
- Respect people's time by keeping interactions relevant and concise

**Unacceptable:**

- Harassment, personal attacks, or discriminatory language
- Deliberate disruption of discussions or project work
- Sharing private information (emails, addresses, etc.) without consent
- Sustained patterns of behavior that make the project hostile to participate in

## Scope

This applies to all project spaces: issues, pull requests, code reviews, and any other channel associated with this project.

## Enforcement

The goal of enforcement is to protect the ability of contributors to work effectively — not to police opinions or communication styles.

If something is a problem, contact the project maintainers. Concerns will be handled discreetly and proportionally. We assume good faith and address patterns of behavior, not isolated moments taken out of context.
